# 鹅的笔记设计系统

## 设计方向

- 场景是一张在白天与傍晚都能长时间使用的安静书桌。
- 原生壳退到背景，编辑纸面保持最清晰的对比度。
- 视觉保持克制、窄侧栏、稳定内容宽度和少量珊瑚色提示，不使用假标题栏或低对比度文字。
- 系统控件保持熟悉。个性来自空间、排版和状态节奏。

## 令牌体系

- `Design/tokens.json` 是颜色、间距、圆角、字体和动效的唯一来源。
- 颜色使用语义名称，不在组件中写品牌色常量。
- 浅色和深色共享相同的层级关系，不做机械反色。
- 内容文字与背景对比度至少为 4.5:1。
- 大号文字与背景对比度至少为 3:1。
- 焦点色只用于键盘焦点和当前操作，不作装饰。

## 表面层级

- `canvas` 是编辑纸面。
- `sidebar` 是导航壳层。
- `surface-muted` 是悬停、选中和次级面板。
- `separator` 只承担结构分隔。
- 静止表面不使用宽而软的装饰投影。

## 排版

- 系统界面使用 SF Pro 与苹方回退。
- 正文默认 16px，行高 1.72，阅读宽度 68ch。
- 页面标题使用 36px、650 字重和紧凑行高。
- 小标签不使用全大写或夸张字距。
- 编辑器可切换系统无衬线、衬线和等宽正文，但原生界面字体不跟随。

## 原生导航

- 侧边栏宽度默认 248pt，可在 216pt 到 340pt 之间调整并恢复。
- 打开文件夹后，侧边栏显示其中的 Markdown 文件和相对路径，并高亮当前文件。
- 未打开文件夹时，侧边栏显示当前会话单独打开的 Markdown 文件。
- 新建、打开文件与打开文件夹必须使用系统文件面板，不维护应用私有笔记库。
- 多文件编辑使用应用内标签页，禁用系统窗口标签页。

## 编辑器

- 文件图标、文件名和正文在同一阅读列内。
- 无法由 BlockNote 无损往返的 Markdown 使用清晰提示的源码编辑器，避免静默改写原文。
- 标题与正文之间不放装饰线。
- 格式工具栏和斜杠菜单只在需要时出现。
- 编辑器正文长度保持 65 到 75 个字符。
- 图片、表格、代码和引用块使用语义边界，不堆叠卡片。
- 右键菜单保留拼写、替换、服务、复制与粘贴。

## 状态与反馈

- 保存中使用低干扰进度文案。
- 已保存使用短暂确认，不常驻抢占注意力。
- 保存失败使用明确错误色、原因和重试入口。
- 空状态解释如何新建或打开本地 Markdown 文件。
- 加载使用就地骨架，不在内容中央放转圈动画。
- 删除、清空和覆盖冲突必须使用原生确认。

## 辅助功能

- 所有按钮有可读标题和工具提示。
- 原生文件列表保持完整键盘和 VoiceOver 语义。
- Web 编辑器保留可见 `:focus-visible` 状态。
- 遵循减少动态、增加对比度和减少透明度偏好。
- 输入法组合期间不触发全局快捷键或自动保存切页。

## 动效

- 状态过渡使用 120ms 到 220ms 的 ease-out。
- 页面加载不做编排式动画。
- 减少动态时取消位移与缩放，仅保留必要的瞬时或淡入反馈。
